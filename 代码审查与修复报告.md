## 二、做得好的地方 👍

- **动态路由设计合理**：`router/dynamicRoutes.ts` 将后端菜单树拍平为 `Layout` 平级子路由，规避了嵌套路由路径重复拼接问题，侧边栏由 `menus` 树自行渲染，解耦清晰。
- **请求层符合契约**：`utils/request.ts` 的响应拦截（`code===0` 直接取 `data`、401 登出、403/其他 toast）与文档 §1.3 完全一致。

- **复用度高**：`ProTable / ProDialog / Chart / DictTag / AmapContainer / OrgPicker / FileUpload / QrcodeViewer` 八件套齐全，列表页普遍使用 `useTable` 组合式函数，样板代码大幅减少。

## 三、发现的问题与修复

### 🔴  blockers：无（无编译错误、无运行时崩溃路径）

### 🟡 需要修复（已修复）

**1. GIS 地图分布图脱离了接口契约（`src/views/gis/GisMap.vue`）**
- 原实现：页面里**硬编码了 2 个点位**，全程未调用 `getAssetPoints / getHeatmap / getRegionTree`，区域下钻、图层切换、类型筛选均未生效——模块 2 等于一个空壳。
- 修复：改为挂载时拉取资产点位 / 区域树；顶部图层切换触发 `getHeatmap` 并展示热力采样量；新增区域树下钻（按名称匹配点位）、资产类型筛选、右侧点位列表（点名称可下钻资产详情）。未配置高德 Key 时通过右侧列表+区域树优雅降级，保证功能可见。

**2. 资产全生命周期缺「状态机流转」（`AssetList.vue` / `AssetDetail.vue`）**
- 文档 §模块4 明确要求：台账行操作含「状态变更」，详情页有「状态机流转弹窗（待启用→在用→改造/调拨→待处置→处置/报废），每次变更填原因」。原代码两个页面都**没有**该能力。
- 修复：
  - `AssetList` 行操作新增「状态变更」按钮（受 `asset:edit` 权限控制）→ 弹窗选目标状态 + 填原因 → `updateAssetStatus`。
  - `AssetDetail` 头部新增「变更状态」按钮 → 同样的弹窗，提交后刷新资产详情。生命周期选项与文档枚举一致（PENDING/IN_USE/VACANT/LEASED/RENOVATING/TO_DISPOSE/SCRAPPED）。

**3. 资产分页 Mock 未切片（`mock/index.ts`）**
- 原 `/api/asset/v1/asset/page` 直接返回 `list: f, total: f.length`，**没有按 pageNum/pageSize 切片**，导致分页「看着能点、实际一页到底」。其余端点都用 `toPage` 切片，此处不一致。
- 修复：改用 `toPage` 切片，并支持 `assetType / status / orgId` 服务端过滤，与 `useTable` 传参对齐。

**4. 字典标签无颜色（`src/components/DictTag/index.vue`）**
- 文档要求「枚举→颜色标签（如资产状态）」。但 `dict/items` 仅返回 `{label,value}`，组件读 `item.type` 永远为空 → **所有标签都是灰色**，状态/类型无法一眼区分。
- 修复：在组件内增加「枚举值→el-tag 颜色」映射（资产状态/合同状态/账单状态/隐患等级/启用停用等），若字典项自带 `type` 则优先，满足颜色标签要求。

### 💭 优化项（本次未改，建议后续处理）

- **QrcodeViewer 为文本占位**：当前把编码打印成文字框。建议引入 `qrcode` 库生成真实二维码（一资产一码）。
- **AssetDetail 关联 Tab 为静态空态**：关联权证 / 租赁合同 / 运维记录 / 操作日志目前是 `el-empty`，待后端提供跨模块接口后对接。
- **流程设计器 FlowDesigner 占位**：依赖待评审项 F01（bpmn-js vs 自研），属预期内。
- **AmapContainer 未真正渲染底图**：依赖待评审项 F02（地图厂商）与 `VITE_APP_AMAP_KEY`；缺 Key 时显示占位，合理。
- **`/api/asset/v1/asset/page/export`（POST）为多余端点**：与文档 `POST /asset/v1/asset/export` 重复，且无前端调用，可清理。
