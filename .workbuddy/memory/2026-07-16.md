## 资产台账接口适配（根据接口文档适配前端）

### 接口文档信息
- 文件：`C:\Users\Renata\Downloads\不动产管理-接口文档.html`
- 后端地址：`http://localhost:8081`
- 三个端点：
  - `GET /api/v1/assets` — 资产列表分页查询（params: page, pageSize, buildingName, status[int 0-闲置/1-在用/2-欠费风险], source）
  - `GET /api/v1/assets/{id}` — 资产详情（id 为 int64）
  - `GET /api/v1/assets/statistics` — 资产统计（返回 totalCount, totalValue, inUseCount, overdueRiskCount）

### AssetVO 字段
id, buildingName, ownerUnit, propertyNo(产权证号), address, usageTerm(使用期限), buildingArea, landNature, structureType, source, assetValue, statusName(使用状态名称)

### 响应结构
- 列表：`{ code:0, data: { list: AssetVO[], pagination: { page, total, pageSize } } }`（无 requestId）
- 详情：`{ code:0, data: AssetVO }`
- 统计：`{ code:0, data: { totalCount, totalValue, inUseCount, overdueRiskCount } }`

### 适配策略
**适配层模式** — 在 `src/api/asset/index.ts` 做参数转换 + 响应拍平，useTable 和页面组件基础设施不变：
- 参数：`pageNum→page`，`useStatus`(string) → `status`(int)
- 响应：嵌套 `{ list, pagination: { total } }` → 扁平 `{ list, total }`
- 统计：`totalCount→total, inUseCount→inUse, overdueRiskCount→arrearsRisk` + 在用率计算

## 资产台账 Mock 快速切换功能

### 需求
后端不可连时，快速切换到前端 mock 数据，无需启动后端服务。

### 方案
在 API 适配层添加环境变量开关，不依赖 vite-plugin-mock（避免 .mjs 缓存问题）：
- 新增 `VITE_USE_ASSET_MOCK=false` 到 `.env.development`
- 新建 `src/api/asset/mock-data.ts` — 9 条 AssetVO 格式样例数据，支持分页/筛选/详情/统计
- 修改 `src/api/asset/index.ts` — 3 个核心函数（getAssetPage/getAssetDetail/getAssetLedgerStat）在 flag=true 时走 mock 路径
- 更新 `src/env.d.ts` — 添加 VITE_USE_ASSET_MOCK 和 VITE_API_PROXY_TARGET 类型声明
- mock 数据返回格式与真实 API 一致（ApiPageResult/AssetVO/AssetStatisticsVO），适配层转换逻辑复用
- 控制台输出橙色提示信息便于确认当前模式

### 使用方式
后端不可用时：把 `.env.development` 中 `VITE_USE_ASSET_MOCK` 改为 `true`，重启 dev server 即可。
